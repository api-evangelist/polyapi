<p>In <strong><a href="https://polyapi.io/tag/release-20/">Release 20</a></strong>, we introduced API support for function states and owner relationships. With <strong><a href="https://polyapi.io/tag/release-21/">Release 21</a></strong>, we’ve expanded on this by adding <strong>UI support</strong> for both capabilities, enabling more intuitive and collaborative development workflows.</p>



<p>These enhancements help teams better coordinate by making it clear <strong>what state a function is in</strong>—such as ALPHA, BETA, or STABLE—and <strong>who the designated owner is</strong>. This structure has now been extended across the platform to include all object types, such as <strong>variables, webhooks</strong>, and <strong>all function types</strong>.</p>



<h2 class="wp-block-heading">What’s New in R21</h2>



<ul class="wp-block-list">
<li><strong>Function Ownership</strong>: All function types can now be associated with individual users. While currently this is informational, we’re laying the groundwork for future features like:
<ul class="wp-block-list">
<li>Better representation of the relationship visually with one click navigation</li>



<li>Notifications about schema discrepancies</li>



<li>Usage alerts</li>



<li>Approval workflows</li>



<li>Streamlined owner assignment during deployment, training, and creation<br /></li>
</ul>
</li>



<li><strong>Function Lifecycle States</strong>: Every function now includes a lifecycle state indicator. Learn more about these states and their long-term significance here:<a href="https://polyapi.io/function-lifecycle-states/"> Function Lifecycle States</a></li>
</ul>
